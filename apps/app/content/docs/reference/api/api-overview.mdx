Deepcrawl exposes the same typed primitives across the Node SDK and the REST
workers that power `https://api.deepcrawl.dev`.

---

## Shared APIs Options types

Import them directly from `deepcrawl/types` when you need deeper control:

- `ScrapeOptions`, `FetchOptions`, `SafeHeaders`, `ScrapedData`, `MetaFiles` -
  canonical scrape primitives .
- `MetadataOptions`, `PageMetadata` - fine grained metadata toggles and
  payload.
- `LinkExtractionOptions`, `ExtractedLinks`, `LinksTree` - tree traversal and
  link categorization helpers.
- `ReaderCleaningOptions`, `HTMLRewriterOptions` - switch between the
  cheerio-reader and HTMLRewriter processors.
- `CacheOptions`, `MetricsOptions`, `Metrics` - shared cache and instrumentation
  types.


## Error payloads

Deepcrawl contracts surface typed error bodies so you can branch on them:

- `READ_ERROR_RESPONSE` -> `ReadErrorResponse`.
- `LINKS_ERROR_RESPONSE` -> `LinksErrorResponse`.
- `RATE_LIMITED` -> `{ operation: string; retryAfter: number }`.
- `LOGS_INVALID_DATE_RANGE` and `LOGS_INVALID_SORT` describe validation
  failures on logs filters.
- `INVALID_EXPORT_FORMAT` is returned when `format` does not match the stored
  response.

Use the types above to narrow errors in SDK catches or REST clients.
